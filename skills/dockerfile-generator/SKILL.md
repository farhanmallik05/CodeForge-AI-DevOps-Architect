# Dockerfile Generator

You are the second skill in the CodeForge pipeline. You receive a **Stack Profile** from `stack-analyzer` and generate a production-grade, multi-stage Dockerfile plus a matching `.dockerignore` file.

## Core Principles (Non-Negotiable)

1. **Multi-stage builds** — Always. No exceptions. Builder stage for dependencies/compilation, production stage with minimal base.
2. **Non-root execution** — Every Dockerfile must switch to a non-root user before `CMD`.
3. **HEALTHCHECK instruction** — Required in every Dockerfile.
4. **Layer cache optimization** — Copy dependency manifests first, install, then copy source.
5. **Minimal base images** — Alpine, Distroless, or Scratch for production stages.
6. **OCI labels** — Include standard metadata labels on every image.
7. **No secrets in build** — Never accept secrets via `ARG` or `COPY`.

## Language-Specific Templates

### Node.js (Express, NestJS, Fastify, Next.js)

```dockerfile
# ============================================
# Generated by CodeForge — AI DevOps Architect
# Stack: Node.js / Express
# ============================================

# --- Stage 1: Build ---
FROM node:20.11.0-alpine3.19 AS builder
WORKDIR /app

# Layer cache: install dependencies first
COPY package.json package-lock.json ./
RUN npm ci --ignore-scripts

# Copy source and build
COPY . .
RUN npm run build

# Prune dev dependencies
RUN npm ci --omit=dev --ignore-scripts

# --- Stage 2: Production ---
FROM node:20.11.0-alpine3.19 AS production
WORKDIR /app

# Security: create non-root user
RUN addgroup -g 1001 -S appgroup && \
    adduser -u 1001 -S appuser -G appgroup

# Copy only production artifacts
COPY --from=builder --chown=appuser:appgroup /app/package.json ./
COPY --from=builder --chown=appuser:appgroup /app/node_modules ./node_modules
COPY --from=builder --chown=appuser:appgroup /app/dist ./dist

# OCI labels
LABEL org.opencontainers.image.source="https://github.com/OWNER/REPO"
LABEL org.opencontainers.image.description="Production image for APPNAME"
LABEL org.opencontainers.image.licenses="MIT"

# Security: switch to non-root
USER appuser

# Expose application port
EXPOSE 3000

# Health check
HEALTHCHECK --interval=30s --timeout=5s --start-period=10s --retries=3 \
  CMD wget --no-verbose --tries=1 --spider http://localhost:3000/health || exit 1

# Start application
CMD ["node", "dist/server.js"]
```

### Python (FastAPI / Django / Flask)

```dockerfile
# ============================================
# Generated by CodeForge — AI DevOps Architect
# Stack: Python / FastAPI
# ============================================

# --- Stage 1: Build ---
FROM python:3.12-slim AS builder
WORKDIR /app

# Install build dependencies
RUN pip install --no-cache-dir --upgrade pip

# Layer cache: install requirements first
COPY requirements.txt ./
RUN pip install --no-cache-dir --prefix=/install -r requirements.txt

# --- Stage 2: Production ---
FROM python:3.12-slim AS production
WORKDIR /app

# Security: non-root user
RUN groupadd -r -g 1001 appgroup && \
    useradd -r -u 1001 -g appgroup -s /sbin/nologin appuser

# Copy installed packages from builder
COPY --from=builder /install /usr/local

# Copy application source
COPY --chown=appuser:appgroup . .

LABEL org.opencontainers.image.source="https://github.com/OWNER/REPO"

USER appuser
EXPOSE 8000

HEALTHCHECK --interval=30s --timeout=5s --start-period=15s --retries=3 \
  CMD python -c "import urllib.request; urllib.request.urlopen('http://localhost:8000/health')" || exit 1

CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### Go

```dockerfile
# ============================================
# Generated by CodeForge — AI DevOps Architect
# Stack: Go
# ============================================

# --- Stage 1: Build ---
FROM golang:1.22-alpine AS builder
WORKDIR /app

# Layer cache: download modules first
COPY go.mod go.sum ./
RUN go mod download && go mod verify

# Build static binary
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -ldflags="-s -w" -o /app/server ./cmd/server

# --- Stage 2: Production (Scratch) ---
FROM scratch AS production

# Import CA certificates for HTTPS
COPY --from=builder /etc/ssl/certs/ca-certificates.crt /etc/ssl/certs/

# Copy binary
COPY --from=builder /app/server /server

EXPOSE 8080

HEALTHCHECK --interval=30s --timeout=5s --retries=3 \
  CMD ["/server", "--health-check"]

ENTRYPOINT ["/server"]
```

## Adaptation Rules

- **If TypeScript detected:** Add `RUN npm run build` in builder, copy `dist/` to production
- **If no build step:** Skip builder compilation, copy source directly
- **If monorepo:** Generate Dockerfile per workspace with appropriate COPY context
- **If `docker-compose.yml` exists:** Note it but generate standalone Dockerfile (compose is for local dev)
- **Adjust HEALTHCHECK** based on detected health endpoint from Stack Profile
- **Adjust EXPOSE** based on detected port
- **Adjust CMD** based on detected start command
