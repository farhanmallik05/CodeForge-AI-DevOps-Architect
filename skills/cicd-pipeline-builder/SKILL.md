# CI/CD Pipeline Builder

You are the third skill in the CodeForge pipeline. You generate a complete GitHub Actions workflow that takes code from push to production deployment, incorporating security at every stage.

## Pipeline Architecture

Generate a workflow with these stages, each as a separate job with proper dependency chains:

```
┌──────────┐    ┌──────────┐    ┌────────────┐    ┌──────────┐    ┌──────────┐
│   Lint   │───▶│   Test   │───▶│  Security  │───▶│  Build   │───▶│  Deploy  │
│ & Format │    │          │    │    Scan    │    │ & Push   │    │          │
└──────────┘    └──────────┘    └────────────┘    └──────────┘    └──────────┘
```

## Generated Workflow File

Output path: `.github/workflows/ci-cd.yml`

### Workflow Structure

```yaml
# ============================================
# Generated by CodeForge — AI DevOps Architect
# Pipeline: Build, Test, Scan, Deploy
# ============================================
name: CI/CD Pipeline

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

# Restrict default permissions (principle of least privilege)
permissions:
  contents: read

env:
  REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}

jobs:
  # ─── Stage 1: Lint & Format ───
  lint:
    name: Lint & Format Check
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      - run: npm ci
      - run: npm run lint

  # ─── Stage 2: Test ───
  test:
    name: Run Tests
    runs-on: ubuntu-latest
    needs: [lint]
    strategy:
      matrix:
        node-version: [18, 20, 22]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}
          cache: 'npm'
      - run: npm ci
      - run: npm test
      - uses: actions/upload-artifact@v4
        if: always()
        with:
          name: test-results-${{ matrix.node-version }}
          path: coverage/

  # ─── Stage 3: Security Scan ───
  security:
    name: Security Scan
    runs-on: ubuntu-latest
    needs: [test]
    permissions:
      security-events: write
    steps:
      - uses: actions/checkout@v4

      # Dependency audit
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      - run: npm audit --audit-level=high

      # Container vulnerability scan
      - name: Run Trivy vulnerability scanner
        uses: aquasecurity/trivy-action@0.28.0
        with:
          scan-type: 'fs'
          scan-ref: '.'
          format: 'sarif'
          output: 'trivy-results.sarif'
          severity: 'CRITICAL,HIGH'

      - name: Upload Trivy results to GitHub Security
        uses: github/codeql-action/upload-sarif@v3
        if: always()
        with:
          sarif_file: 'trivy-results.sarif'

  # ─── Stage 4: Build & Push ───
  build:
    name: Build & Push Container Image
    runs-on: ubuntu-latest
    needs: [security]
    permissions:
      contents: read
      packages: write
      id-token: write  # For Cosign signing
    outputs:
      image-digest: ${{ steps.build-push.outputs.digest }}
    steps:
      - uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Log in to Container Registry
        uses: docker/login-action@v3
        with:
          registry: ${{ env.REGISTRY }}
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Extract metadata
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}
          tags: |
            type=sha,prefix=
            type=ref,event=branch
            type=semver,pattern={{version}}

      - name: Build and push
        id: build-push
        uses: docker/build-push-action@v6
        with:
          context: .
          push: ${{ github.event_name != 'pull_request' }}
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=gha
          cache-to: type=gha,mode=max
          provenance: true
          sbom: true

      # Generate SBOM
      - name: Generate SBOM
        uses: anchore/sbom-action@v0
        with:
          image: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}@${{ steps.build-push.outputs.digest }}
          format: spdx-json

  # ─── Stage 5: Deploy ───
  deploy:
    name: Deploy to Kubernetes
    runs-on: ubuntu-latest
    needs: [build]
    if: github.ref == 'refs/heads/main' && github.event_name != 'pull_request'
    environment:
      name: production
      url: https://app.example.com
    permissions:
      contents: read
      id-token: write
    steps:
      - uses: actions/checkout@v4

      - name: Set up kubectl
        uses: azure/setup-kubectl@v4

      - name: Deploy to cluster
        run: |
          kubectl set image deployment/app \
            app=${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}@${{ needs.build.outputs.image-digest }} \
            --namespace production
```

## Adaptation Rules

### Language-Specific Steps

Adjust the `lint` and `test` jobs based on the Stack Profile:

| Language | Setup Action | Install | Lint | Test |
|---|---|---|---|---|
| Node.js | `actions/setup-node@v4` | `npm ci` | `npm run lint` | `npm test` |
| Python | `actions/setup-python@v5` | `pip install -r requirements.txt` | `ruff check .` | `pytest` |
| Go | `actions/setup-go@v5` | `go mod download` | `golangci-lint run` | `go test ./...` |
| Java | `actions/setup-java@v4` | (Maven/Gradle auto) | `./mvnw checkstyle:check` | `./mvnw test` |
| Rust | `dtolnay/rust-toolchain@stable` | (cargo auto) | `cargo clippy` | `cargo test` |

### Caching Strategy

| Package Manager | Cache Config |
|---|---|
| npm | `cache: 'npm'` in setup-node |
| yarn | `cache: 'yarn'` in setup-node |
| pnpm | Use `pnpm/action-setup@v4` + cache |
| pip | `cache: 'pip'` in setup-python |
| Go modules | `cache: true` in setup-go |

### Security Scanning Variants

- **Container images:** Trivy with `image` scan type after build
- **Filesystem/dependencies:** Trivy with `fs` scan type before build
- **SAST:** CodeQL analysis for supported languages
- **Secrets detection:** TruffleHog or gitleaks as a pre-job step

### Branch Protection

- `main` branch triggers: lint → test → security → build → deploy
- `develop` and feature branches: lint → test → security → build (no deploy)
- Pull requests: lint → test → security (no build/deploy)
