## Prometheus ServiceMonitor

```yaml
# ============================================
# Generated by CodeForge — AI DevOps Architect
# Prometheus ServiceMonitor for APP_NAME
# ============================================
apiVersion: monitoring.coreos.com/v1
kind: ServiceMonitor
metadata:
  name: APP_NAME
  namespace: APP_NAME
  labels:
    app.kubernetes.io/name: APP_NAME
    app.kubernetes.io/managed-by: codeforge
    release: prometheus  # Matches Prometheus Operator selector
spec:
  selector:
    matchLabels:
      app.kubernetes.io/name: APP_NAME
  namespaceSelector:
    matchNames:
      - APP_NAME
  endpoints:
    - port: http
      path: /metrics
      interval: 30s
      scrapeTimeout: 10s
      honorLabels: true
```

## Alerting Rules

```yaml
# ============================================
# Generated by CodeForge — AI DevOps Architect
# Prometheus Alerting Rules for APP_NAME
# ============================================
apiVersion: monitoring.coreos.com/v1
kind: PrometheusRule
metadata:
  name: APP_NAME-alerts
  namespace: APP_NAME
  labels:
    app.kubernetes.io/name: APP_NAME
    app.kubernetes.io/managed-by: codeforge
    release: prometheus
spec:
  groups:
    - name: APP_NAME.availability
      rules:
        # CRITICAL: Pod is crash-looping
        - alert: PodCrashLooping
          expr: |
            rate(kube_pod_container_status_restarts_total{
              namespace="APP_NAME",
              pod=~"APP_NAME-.*"
            }[15m]) > 0
          for: 5m
          labels:
            severity: critical
            team: platform
          annotations:
            summary: "Pod {{ $labels.pod }} is crash-looping"
            description: "Pod {{ $labels.pod }} in namespace {{ $labels.namespace }} has been restarting for more than 5 minutes."
            runbook_url: "https://runbooks.example.com/PodCrashLooping"

        # CRITICAL: No pods running
        - alert: DeploymentReplicasUnavailable
          expr: |
            kube_deployment_status_replicas_available{
              namespace="APP_NAME",
              deployment="APP_NAME"
            } == 0
          for: 1m
          labels:
            severity: critical
            team: platform
          annotations:
            summary: "Deployment APP_NAME has zero available replicas"
            description: "All replicas of APP_NAME are unavailable. Service is completely down."

    - name: APP_NAME.performance
      rules:
        # WARNING: High error rate
        - alert: HighErrorRate
          expr: |
            (
              sum(rate(http_requests_total{
                namespace="APP_NAME",
                code=~"5.."
              }[5m]))
              /
              sum(rate(http_requests_total{
                namespace="APP_NAME"
              }[5m]))
            ) > 0.05
          for: 5m
          labels:
            severity: warning
            team: backend
          annotations:
            summary: "High HTTP 5xx error rate for APP_NAME"
            description: "More than 5% of requests are returning 5xx errors. Current rate: {{ $value | humanizePercentage }}."

        # WARNING: High latency
        - alert: HighRequestLatency
          expr: |
            histogram_quantile(0.95,
              sum(rate(http_request_duration_seconds_bucket{
                namespace="APP_NAME"
              }[5m])) by (le)
            ) > 1.0
          for: 10m
          labels:
            severity: warning
            team: backend
          annotations:
            summary: "P95 latency exceeding 1s for APP_NAME"
            description: "The 95th percentile request latency is {{ $value }}s, exceeding the 1s SLO threshold."

    - name: APP_NAME.resources
      rules:
        # WARNING: High CPU usage
        - alert: HighCPUUsage
          expr: |
            (
              sum(rate(container_cpu_usage_seconds_total{
                namespace="APP_NAME",
                container="APP_NAME"
              }[5m]))
              /
              sum(kube_pod_container_resource_limits{
                namespace="APP_NAME",
                container="APP_NAME",
                resource="cpu"
              })
            ) > 0.85
          for: 15m
          labels:
            severity: warning
            team: platform
          annotations:
            summary: "High CPU utilization for APP_NAME"
            description: "CPU usage is above 85% of limits for more than 15 minutes. Consider increasing CPU limits or scaling horizontally."

        # WARNING: High memory usage (OOM risk)
        - alert: HighMemoryUsage
          expr: |
            (
              sum(container_memory_working_set_bytes{
                namespace="APP_NAME",
                container="APP_NAME"
              })
              /
              sum(kube_pod_container_resource_limits{
                namespace="APP_NAME",
                container="APP_NAME",
                resource="memory"
              })
            ) > 0.85
          for: 10m
          labels:
            severity: warning
            team: platform
          annotations:
            summary: "High memory usage for APP_NAME — OOM kill risk"
            description: "Memory usage is above 85% of limits. Risk of OOM kill. Current usage: {{ $value | humanizePercentage }}."
```

## Grafana Dashboard Overview

Generate a Grafana dashboard JSON Model with these panels arranged in rows:

### Row 1: Request Metrics (RED Method)
| Panel | Visualization | Query |
|---|---|---|
| Request Rate | Stat + Sparkline | `sum(rate(http_requests_total{namespace="APP_NAME"}[5m]))` |
| Error Rate (%) | Gauge (red threshold >5%) | `sum(rate(http_requests_total{code=~"5.."}[5m])) / sum(rate(http_requests_total[5m]))` |
| P50 Latency | Stat | `histogram_quantile(0.5, sum(rate(http_request_duration_seconds_bucket[5m])) by (le))` |
| P95 Latency | Stat | `histogram_quantile(0.95, ...)` |
| P99 Latency | Stat | `histogram_quantile(0.99, ...)` |

### Row 2: Resource Usage
| Panel | Visualization | Query |
|---|---|---|
| CPU Usage vs Limit | Time series (fill between request/limit) | `container_cpu_usage_seconds_total` |
| Memory Usage vs Limit | Time series | `container_memory_working_set_bytes` |
| Network I/O | Time series (rx/tx) | `container_network_receive_bytes_total` / `transmit` |

### Row 3: Pod Health
| Panel | Visualization | Query |
|---|---|---|
| Pod Status | Table | `kube_pod_status_phase` |
| Restart Count | Stat (red if >0) | `kube_pod_container_status_restarts_total` |
| HPA Status | Stat | `kube_horizontalpodautoscaler_status_current_replicas` |

## Structured Logging Guidelines

Generate a `logging-config.md` documenting:

1. **JSON log format** — All application logs must be structured JSON:
   ```json
   {
     "timestamp": "2024-01-15T10:30:00.000Z",
     "level": "info",
     "message": "Request processed",
     "service": "APP_NAME",
     "trace_id": "abc123",
     "span_id": "def456",
     "method": "GET",
     "path": "/api/data",
     "status": 200,
     "duration_ms": 42
   }
   ```

2. **Correlation ID propagation** — Extract `X-Request-ID` or `X-Correlation-ID` from incoming headers, propagate to all downstream calls, and include in every log entry.

3. **Log levels** — Use consistent levels: `debug`, `info`, `warn`, `error`, `fatal`.

4. **Sensitive data** — Never log passwords, tokens, credit card numbers, or PII. Mask or redact.

## Adaptation Rules

- Replace `APP_NAME` with the actual application name from Stack Profile
- If the application doesn't expose `/metrics`, add guidance for instrumenting with the appropriate Prometheus client library (e.g., `prom-client` for Node.js, `prometheus_client` for Python)
- Adjust alert thresholds based on application type (API vs worker vs cron)
- For stateful applications, add additional alerts for PVC usage and storage pressure
